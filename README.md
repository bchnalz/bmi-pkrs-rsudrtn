## Environment Setup

1. Copy `.env.example` into `.env`
2. Fill:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   - `OPENAI_API_KEY` (server-side, for Vercel/API function)
   - `OPENAI_MODEL` (optional, defaults to `gpt-4o-mini`)
   - `SUPABASE_URL` (server-side for API routes; usually same as VITE_SUPABASE_URL)
   - `SUPABASE_SERVICE_ROLE_KEY` (server-side; required for food_items and food_logs write operations)

## Supabase SQL

Run this SQL in Supabase SQL Editor:

```sql
create table if not exists public.bmi_submissions (
  id bigint generated by default as identity primary key,
  name text not null,
  instansi text not null,
  phone_number text not null,
  gender text not null,
  weight_kg numeric not null,
  height_cm numeric not null,
  age integer not null,
  bmi numeric not null,
  category text not null,
  created_at timestamptz not null default now()
);
```

If Row Level Security is enabled, add policies so insert/select can run with your chosen auth model.
